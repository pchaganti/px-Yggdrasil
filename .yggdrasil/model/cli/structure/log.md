## [2026-05-29T06:27:03.177Z]
Added ctx-graph.ts implementing createCtxGraph and UndeclaredGraphReadError. The structure-aspect runtime needs a guarded graph surface so that check.mjs authors can only read nodes that are structurally related to the current node (direct relations, hierarchy ancestors, own descendants, and transitively the descendants of relation targets). Without this guard, a check.mjs could accidentally read arbitrary graph nodes, producing coupling outside declared dependencies and making enforcement unreliable. The allowed-node-id computation is deterministic: it walks declared relations (and their subtrees), the ancestor chain, and the current node's own subtree — all from the in-memory graph, no I/O.

## [2026-05-29T14:09:41.987Z]
The structure-aspect runtime exposes a ctx object to untrusted check.mjs aspect code, whose filesystem and graph reads must be confined to a declared allow-set so an aspect cannot exfiltrate files or graph metadata outside its scope. Three confinement gaps were closed. First, the JSON/YAML/TOML parser helpers accepted a raw string path and read it with no allow-set check at all, so an aspect could parse any file on disk (for example a system password file or any absolute path); the string-path branch now goes through the same allow-set-enforcing resolver the filesystem surface uses. Second, the filesystem guard matched allowed paths by prefix but never collapsed parent-directory (..) segments and never re-checked that the resolved path stayed inside the repository, so a path that began with an allowed directory followed by enough .. segments resolved outside the repository and was read; resolution now normalizes the path, rejects absolute inputs, and rejects any path that escapes the repository root before applying the allow-set. Third, one graph accessor that returns a node's outgoing relations skipped the allow-set assertion that the node and children accessors enforce, letting an aspect read relation metadata of arbitrary out-of-scope nodes by passing a hand-built node identifier; that accessor now asserts the node is in scope first. All read paths now funnel through a single escape-safe, allow-set-enforcing resolver shared between the filesystem and parser surfaces so the two cannot drift apart, and the relation accessor enforces the same scope check as its siblings.
## [2026-05-29T18:15:39.894Z]
ctx.parseAst called on a file not pre-warmed by the dispatcher previously threw a plain Error that fell through to the generic STRUCTURE_CHECK_THROWN hard runner error, making it indistinguishable from an arbitrary check.mjs crash. The fix introduces a typed ParseAstNotPrewarmedError class in ctx-parsers.ts; the runner's catch block detects it via instanceof and converts it to a structure-aspect-parseast-not-prewarmed violation (succeeded: false), matching the pattern already established for UndeclaredFsReadError and UndeclaredGraphReadError. This surfaces actionable guidance to the author rather than a hard crash, and keeps parity across all three guard-error kinds.
