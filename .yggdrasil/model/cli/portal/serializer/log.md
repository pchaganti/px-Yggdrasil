The static page now inlines a SET of focused frontend modules in a fixed dependency order, not a single bootstrap script. A single offline HTML file has no module system, so the modules cannot use import/require; instead each attaches to one shared browser global and they are concatenated as a plain script sequence — the namespace first, the leaf modules next, the shell and dispatcher that consume them, and the bootstrap last because it orchestrates the rest. The serializer owns that ordering: the order is the contract that keeps a later module from referencing an earlier one before it is defined. Modules are separated by a newline on concatenation so a file ending without one cannot fuse into the next file first line. The shell template placeholder for the old single bootstrap was replaced by one modules placeholder the serializer fills with the concatenation, keeping all template-asset assembly in this one node.
