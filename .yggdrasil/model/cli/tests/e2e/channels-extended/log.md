Adds end-to-end coverage for how a conditional applicability filter behaves when a rule reaches a component indirectly — inherited from an ancestor, supplied as a type default, or pulled in by another rule it references. The filter must be evaluated against the receiving component itself, not the place the rule was declared, and the strictest enforcement level must win when a rule arrives by several routes at once. A mistake here either applies a rule to components it was scoped away from or silently drops it; proving the indirect-arrival cases against the real binary closes that gap.
