## [2026-05-29T06:06:39.257Z]
Added collectAllowedReadsForAspect, which computes the set of repo-relative paths a structure aspect on a given node is allowed to read. The allowed set is composed from four sources: the node's own mapping (minus paths exclusively claimed by immediate child nodes when the parent selectively enumerates sibling files), relation target mappings and their transitive descendants, ancestor node mappings, and descendant node mappings. This pure-sync function reuses GraphNode object references from the loaded Graph to avoid redundant string-to-node lookups beyond relation target resolution, satisfying the single-source-graph-queries design requirement.

## [2026-05-29T06:27:03.177Z]
Added ctx-graph.ts implementing createCtxGraph and UndeclaredGraphReadError. The structure-aspect runtime needs a guarded graph surface so that check.mjs authors can only read nodes that are structurally related to the current node (direct relations, hierarchy ancestors, own descendants, and transitively the descendants of relation targets). Without this guard, a check.mjs could accidentally read arbitrary graph nodes, producing coupling outside declared dependencies and making enforcement unreliable. The allowed-node-id computation is deterministic: it walks declared relations (and their subtrees), the ancestor chain, and the current node's own subtree — all from the in-memory graph, no I/O.
