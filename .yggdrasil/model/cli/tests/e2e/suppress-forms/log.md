Adds end-to-end coverage for the inline waiver syntax as honored by the local deterministic reviewers. A waiver is an explicit, human-authored exception that tells the reviewer to accept code that would otherwise violate a rule; its scoping rules must be exact, because an over-broad waiver silently disables enforcement and an under-broad one fails to cover the intended code. This suite spawns the real binary and pins the deterministic-reviewer scoping precisely: an unterminated block-disable waives through end of file, a single-line wildcard waives all rules on the following line, a misspelled rule id waives nothing, the correctly-spelled id waives, a named block waives only its own rule, and the graph-aware reviewer honors the same markers as the single-file one. A separate observation — that the deterministic reviewers scope waivers strictly by line whereas the language-model reviewer is instructed to scope them contextually to the surrounding construct, a difference the published syntax guide does not yet spell out — is recorded for follow-up; the tests pin the actual deterministic behavior.
