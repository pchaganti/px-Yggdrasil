Adds end-to-end coverage proving a rule's enforcement level behaves correctly when the rule reaches a component indirectly — inherited from its component type, from an ancestor type, or via a business-process group. The guarantee is that an advisory rule arriving by any of these routes only warns and never blocks, that a dormant rule arriving by them stays inert, and that when the same rule arrives by several routes the strictest level wins. A mistake here would either block a build on a rule meant only to advise, or silently run a rule meant to be dormant; this proves it against the real binary.
