## [2026-05-31T17:27:18.460Z]
Moved the shared command-layer support helpers — the graph-load-or-abort wrapper and the unexpected-error funnel — out of the formatters layer and into the command layer. These helpers must reach both the engine (to load the graph) and the formatters (to build the uniform what/why/next message), and only the command layer may legally depend on both; keeping them in the formatters layer was an upward dependency on the engine that the layering rules forbid. The helpers register no command of their own, so they live under a dedicated command-support classification rather than as a command handler. Command handlers now import them from their new command-layer location.
## [2026-05-31T21:13:17.658Z]
Added OutdatedSchemaVersionError handling to loadGraphOrAbort. When the loader detects an on-disk graph version older than the CLI supports (introduced by the new lower-bound gate in graph-loader), preamble now catches this specific error type and renders a clean what/why/next message — what: the graph is at version X, older than this CLI; why: the CLI reads only the current on-disk format, not legacy ones; next: run yg init --upgrade to migrate, then re-run. This keeps the CLI's error surface consistent: every user-visible failure from graph loading is a structured three-part message, not a raw exception stack trace. The symmetric pattern mirrors the existing UnsupportedSchemaVersionError handler for the upper-bound case.
