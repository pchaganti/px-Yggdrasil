## [2026-06-08T08:24:05.118Z]
Added two new tests for tiered coverage behavior in runCheck. The first test verifies that the default config (no coverage block in yg-config.yaml, which defaults to required: ['/']) still produces a single unmapped-files error and no uncovered-advisory warnings, preserving backward compatibility. The second test verifies that a scoped config with a coverage.required root and an excluded root correctly splits uncovered files: files under the required root that are not mapped produce an unmapped-files error, files outside any required root produce an uncovered-advisory warning, and files under the excluded root are silently dropped. The test uses an exact-file mapping (not a directory mapping) for the node so that a sibling file in the same directory remains uncovered.
