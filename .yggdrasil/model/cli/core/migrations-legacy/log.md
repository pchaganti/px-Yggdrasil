## [2026-05-31T16:03:34.061Z]
Replaced the hand-inlined path-separator normalization with calls to a single shared helper. The same small idiom — convert backslash separators to forward slashes, and in most places also strip a trailing slash — had been copied across many modules, so the normalization rule lived in dozens of places at once and any change to it risked drifting them out of step. Consolidating it behind one well-named helper means the rule lives in exactly one spot and each call site reads by intent instead of by a repeated regex. Behavior is unchanged: the helper bodies are byte-for-byte equivalent to the expressions they replace, and the full test suite passes identically.
## [2026-06-01T07:57:54.243Z]
Stopped these legacy migrations from writing the project version themselves. A single writer now owns the version field of the project config: the migration runner advances it one step per successfully-completed migration. Previously each migration also wrote the same version string at its end, a redundant second write that duplicated the runner's job and risked bypassing the warnings gate (a migration could advance the version even when it had emitted recoverable warnings). With one writer the version moves exactly when, and only when, the runner judges a migration complete. Each migration still emits an action line announcing that the runner will bump the version to its target, so the user-facing summary stays unambiguous, and each still embeds its own target version as a string literal so the rule that a migration must name the version it targets continues to hold.
