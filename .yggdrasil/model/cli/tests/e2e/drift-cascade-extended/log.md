Adds end-to-end coverage for the remaining ways a change forces re-verification. Re-review must fire not only when a component's own files change but when a file it inspects across a boundary changes, and a batch re-approval must treat each component independently so one failure does not abort the rest. This suite spawns the real binary to prove cross-boundary re-review, batch independence, the cases that legitimately need no fresh history entry, and recovery from a deleted file, so the system neither misses a change that should re-verify nor demands work that a change did not warrant.
