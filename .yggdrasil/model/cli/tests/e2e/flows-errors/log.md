Adds end-to-end coverage for how a malformed business-process group definition is rejected. A group definition with a missing or empty name, a wrong-shaped participant or rule list, or a missing/unparseable/wrong-type definition file must be refused with a clear diagnostic rather than silently ignored or crashing opaquely. Several filesystem cases currently produce a misleading or unclassified error; the suite pins the actual behavior so the diagnostics can be improved without silently regressing, because a group that fails to load would drop its cross-cutting obligations from every participant.
