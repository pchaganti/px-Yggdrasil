## [2026-06-12T11:46:26.150Z]
Added three regression and edge-case tests for the lock-store: (1) a regression test confirming readLock does not throw when a VerdictEntry reason string contains the text '<<<<<<< HEAD' embedded in JSON — this is safe because the conflict-marker detection was updated to use a line-anchored regex; (2) a serializer escaping test confirming that a reason containing double-quotes, newlines, and backslashes is kept on a single line in the serialized file and survives a write+read roundtrip intact; (3) an unknown-field drop test confirming that extra properties beyond the declared VerdictEntry fields are silently omitted by the serializer and do not appear after a roundtrip.
