## [2026-06-21T13:05:50.794Z]
The read-only verification's prompt-size gate measured a companion-bearing aspect by reconstructing its prompt from the stored read-observations, which cannot distinguish the files a companion hook merely read to make its selection from the files it actually injected into the prompt. A hook that read broadly was therefore measured at the size of its whole reachable set and falsely refused as over the prompt-size limit, even though the live assembly injects only a handful of files. The gate now resolves the companion live — the same resolution the approve path performs — and measures only the injected files; a companion that cannot be resolved surfaces a clear diagnostic instead of a generic unverified state. Consequence: a read-only check may now run a companion resolver hook (which only identifies which files to inject, never judges, and calls no reviewer) when a size limit applies; it still never runs a deterministic check or a reviewer.
