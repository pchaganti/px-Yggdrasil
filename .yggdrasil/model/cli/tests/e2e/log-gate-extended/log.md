Adds end-to-end coverage for the rule that a component's history must record why its code changed before that change is accepted. The requirement to record a reason fires on a real code change regardless of how strictly the component's rules are enforced, and must NOT fire for an upstream-only re-review that changed no code; getting this wrong either nags for entries that are not needed or, worse, lets a code change through with no recorded rationale. This suite spawns the real binary to prove the gate's exact triggering conditions and the history-format and addressing guards around it.
