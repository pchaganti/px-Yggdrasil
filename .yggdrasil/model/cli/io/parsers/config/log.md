## [2026-05-28T07:55:21.872Z]
Added parsing of the optional 'references' sub-mapping on tier entries in yg-config.yaml. This field carries per-tier source-file size limits (max_bytes_per_file and max_total_bytes_per_aspect). The parser validates that references is a YAML mapping, that all keys are from the allowed set, and that each numeric value is a positive integer. Unknown keys and non-positive values produce specific error codes (tier-references-unknown-key, tier-references-max-bytes-per-file-invalid, tier-references-max-total-bytes-invalid). The references key was added to the tier allowed-keys set to prevent false config-tier-unknown-key errors. Defaults (64 KiB per file / 256 KiB total) are applied at consumer sites, not in the parser — absent field or absent sub-key means no override.
## [2026-05-29T15:56:59.142Z]
The per-call CLI-provider timeout in yg-config.yaml was documented in seconds but the parser passed the raw number through as milliseconds, causing a 1000x discrepancy for any user who set the field. The parser now multiplies the configured value by 1000 before storing it, so the internal representation is always milliseconds while the user-facing unit matches the documented seconds. The default (no timeout configured) continues to yield undefined, leaving cli-base to apply its 120-second default unchanged. This is a behavior change: any existing config that set timeout expecting to pass raw milliseconds will now get a 1000x longer timeout.

## [2026-05-30T02:29:44.534Z]
The per-node size guard changed from limiting the raw count of mapped source files to limiting the total number of characters in a node's reviewer context — its mapped source files plus the reference files of its effective aspects — and from a non-blocking warning to a blocking error. The motivation is that the reviewer concatenates all of a node's files into each aspect prompt, so an oversized node risks exceeding the model's context window, which deterministically rejects unchanged code near the truncation point; a file-count limit missed this because one very large file could stay under the count while still blowing the prompt. Binary files do not count toward the budget, and a node mapping a single unsplittable generated or binary artifact (a lockfile, an append-only changelog, an image) may carry a documented per-node exemption with a required justification.

## [2026-06-08T19:22:44.093Z]
Two configuration fields needed stricter validation at parse time so misconfiguration fails loudly rather than degrading silently. The maximum-out-edges fan-out threshold must be a positive whole number; a zero, negative, or fractional value is meaningless as a count and is now rejected with a clear message, matching how the per-node character budget is already guarded.

The requirement that a reviewer tier name an explicit endpoint applies only to the generic OpenAI-compatible provider, which has no built-in host and would otherwise fall back to the public hosted endpoint — a silent and potentially surprising destination. Providers that ship a sensible default host, such as the local model runner that defaults to a loopback address, must not be forced to restate an endpoint; demanding one of them wrongly rejected valid single-tier setups. The endpoint requirement is therefore scoped to the one provider that genuinely lacks a safe default.
