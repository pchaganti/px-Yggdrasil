Adds end-to-end coverage for the advanced behaviors of flow-attached rules, which propagate a shared rule to every node participating in a business process. The motivation is that a flow is the mechanism by which a cross-cutting concern reaches a set of collaborating components at once, so its subtler contracts must be proven against the real binary: that a conditional flow rule reaches only the participants it is scoped to, that an advisory flow rule informs without blocking while an enforced one blocks, that a draft flow rule stays dormant, and — most importantly — that changing which rules or which participants a flow declares forces the affected participants to be re-reviewed rather than silently inheriting an unverified rule. This last guarantee is what makes a flow safe to edit: a reviewer must re-confirm every participant whose obligations changed.
