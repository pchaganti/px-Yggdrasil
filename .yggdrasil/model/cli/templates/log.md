## [2026-05-28T20:25:12.328Z]
Added a paragraph to the canonical agent rules content addressing log-entry mimicry. The self-contained content rule was already present and already clear, but real entries across the repository routinely contained ephemeral references — plan-step numbers, design-section markers, file paths in prose — that the rule forbids.

The path through which the rule was violated turned out to be the recommended workflow itself: the rules instruct an agent to read recent entries before editing a node, which exposes the agent to whatever style the node's prior entries used, and in-context mimicry then propagates that style into the new entry even when the rules text contradicts it. The new paragraph names this mimicry vector explicitly, gives concrete examples of patterns to treat as anti-templates, and grants permission to deviate from prior surface style when those entries pre-date or contradict the current rule.

The intent is a small behavioral override at the exact decision point where agents previously inherited the contamination — not a restatement of the underlying self-containment rule, which the agent already saw. Without this addition, every newly-onboarded agent that follows the recommended pre-edit reading workflow inherits the same failure mode the rule was meant to prevent, and the contaminated style perpetuates itself one node-edit at a time.
