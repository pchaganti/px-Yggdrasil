Parallel test workers were sharing a fixture directory namespace. The hash.test.ts file cleans up any directory starting with 'tmp-htf' after each of its own tests, which also deleted the 'tmp-htf-mtime-security-*' directories that this test suite creates mid-run — causing ENOENT failures when the second test tried to access files deleted by a concurrent cleanup in another worker. Fixed by switching to mkdtemp under os.tmpdir() with a unique prefix per process (yg-mtime-exploit/unchanged-<pid>-), ensuring each test run owns its directory exclusively and cleanup is strictly scoped to what the test itself created.
