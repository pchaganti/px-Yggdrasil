## [2026-05-29T06:06:39.257Z]
Added collectAllowedReadsForAspect, which computes the set of repo-relative paths a structure aspect on a given node is allowed to read. The allowed set is composed from four sources: the node's own mapping (minus paths exclusively claimed by immediate child nodes when the parent selectively enumerates sibling files), relation target mappings and their transitive descendants, ancestor node mappings, and descendant node mappings. This pure-sync function reuses GraphNode object references from the loaded Graph to avoid redundant string-to-node lookups beyond relation target resolution, satisfying the single-source-graph-queries design requirement.
