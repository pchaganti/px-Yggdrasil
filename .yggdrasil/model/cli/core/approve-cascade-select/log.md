Moved the cascade-node and cascade-aspect selection logic out of the command wrapper into a dedicated unit. This logic decides which nodes a batch approve must re-verify when an invocation targets a flow or an aspect, and which of a node effective aspects must be re-run after an upstream change. It was already exercised by unit tests, but it lived in a file the coverage measurement deliberately excludes — the thin command wrappers, which are verified end-to-end instead — so its tested behavior was invisible to the coverage metric. Placing the pure selection logic in its own coverage-counted unit makes the metric reflect what is actually tested, while the command wrapper that wires it to the CLI stays excluded as glue. Behavior is unchanged; the command re-exports the helpers so existing callers and tests are unaffected.
