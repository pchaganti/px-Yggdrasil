Parse new 'status' field on yg-aspect.yaml. Accepts values 'draft', 'advisory', or 'enforced' — three-level enforcement model that lets the reviewer skip aspects still in draft and surface advisory ones as warnings rather than blocking errors. Field is optional; absent means the resolver applies the 'enforced' default. Invalid string values return a dedicated 'aspect-status-invalid' parser error pointing the agent at the three valid values and the knowledge topic for the feature. Early-return on validation failure follows the existing parser convention.
