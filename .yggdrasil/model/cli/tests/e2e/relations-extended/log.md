Adds end-to-end coverage for the full vocabulary of typed dependencies between components and how changing them forces re-review. Dependencies come in several kinds, some requiring paired declarations on both endpoints; the architecture controls which kinds are allowed between which component types, and a dependency that merely exists must not silently grant the depender the provider's obligations. This suite spawns the real binary across the dependency-kind matrix, the paired-event rules, forbidden and broken targets, and the re-review triggered by adding or removing a dependency, so neither over- nor under-propagation slips through.
