Adds end-to-end coverage for structural graph-validation failures that were previously only exercised by in-process unit tests. The guarantee under test is that a structurally invalid graph — an aspect reference that escapes its directory or is duplicated, a reviewer specification of the wrong shape, a node placed under a forbidden parent type, a file that a strict type claims but no node maps, a dependency cycle, or an oversized node lacking a documented size exemption — is rejected by the real binary with the precise reason and a blocking exit. Proving this through the spawned process closes the gap between what the validator computes internally and what an adopter actually sees at the command line.
