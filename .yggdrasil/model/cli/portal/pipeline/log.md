The portal headline must never read stricter than the continuous-enforcement reviewer it mirrors. A deterministic verdict of refused on an ADVISORY rule is, by the honesty model, non-blocking signal — it renders as a warning, never a blocking refusal — yet the derivation bucketed pairs by the RAW verdict kind, so an advisory refusal showed up as a blocking refused: it inflated the refused count, reddened its owning node, and double-displayed (counted again under warnings), contradicting a reviewer run that reported no blocking failures.

The derivation now adjusts every displayed pair state by the pair's EFFECTIVE rule status before it reaches any count, node pill, effective-aspect row, or per-rule tally: a refusal on an advisory rule displays as a warning; a refusal on an enforced rule stays a blocking refusal; verified and the not-yet-confirmed states are unchanged. The status is already carried on each verified pair and each effective-aspect row, so this is a single shared transform applied at one seam, not a re-derivation. The blocking-refusal count is therefore enforced refusals only — exactly what the reviewer blocks on — and the advisory refusals move into their own additive bucket so the coverage ledger still accounts for every expected pair without ever painting a non-blocking signal as a blocking failure. The advisory refusal remains visible and still carries the reviewer's stated reason; it is simply shown as the non-blocking warning it is.
