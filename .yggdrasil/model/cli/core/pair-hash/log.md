## [2026-06-12T11:56:37.658Z]
Implements the frozen input-hash contract (spec §3.1) that makes verdict reuse possible. Every (aspect, unit) verdict entry in the lock is keyed by a sha256 hash of a canonical object encoding the aspect id, scope, node path, rule source hash, subject files, verdict token, and kind discriminator ('llm' or 'deterministic'). LLM pairs additionally fold the aspect description, references, and tier config (excluding api_key and timeout). Deterministic pairs fold the observation set (read, list, exists, graph accesses beyond the subject files).

The core design constraint is that producers and verifiers — including across platforms and Yggdrasil versions — must compute byte-identical hashes. This requires code-point key ordering (the standard < comparator on strings) everywhere, never localeCompare which is environment-sensitive. This replaces the latent cross-platform bug in the old tier-identity.ts:25 which used localeCompare. Serialization and ingredient selection are pinned by golden tests in pair-hash-golden.json; changing them requires a deliberate breaking decision.
